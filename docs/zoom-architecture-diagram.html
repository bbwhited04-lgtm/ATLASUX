Zoom Data Flow
Zoom Integration Flow
1. Authorization: User clicks "Connect Zoom" → Redirect to zoom.us/oauth/authorize → User grants access → Callback to /v1/oauth/zoom/callback → Token exchange via zoom.us/oauth/token → Tokens stored encrypted in PostgreSQL
2. API Usage: Backend calls api.zoom.us/v2 with Bearer token → Schedule meetings (POST /users/{userId}/meetings) · Read recordings (GET /meetings/{meetingId}/recordings) · Send chat (POST /chat/users/{userId}/messages)
3. Webhooks: Zoom sends events to /v1/zoom/webhook → HMAC-SHA256 signature verification → Event logged to audit trail → Agent processes event (meeting started/ended, recording available)
4. Token Refresh: Access tokens refreshed automatically using stored refresh_token before expiry
5. Data Handling: No Zoom data stored beyond OAuth tokens and audit log entries. Meeting content processed in-memory and discarded after agent task completion.
Security Controls
Authentication: JWT tokens with tenant-scoped claims · OAuth 2.0 authorization code flow for Zoom
Authorization: Row-level security on all database tables · Role-based access (owner/admin/member)
Secrets: API keys and OAuth tokens encrypted at rest · Never transmitted via email · Compliance stop on secret detection
Audit: Every API call, agent action, and approval decision logged with timestamp, actor, and tenant ID
Webhook Verification: HMAC-SHA256 signature validation on all incoming Zoom webhook events
Network: All traffic over HTTPS/TLS 1.3 · No plaintext communication

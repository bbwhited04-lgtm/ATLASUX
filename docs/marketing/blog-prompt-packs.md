## Revenue-Generating Prompts

### 1. Product Description That Sells
```
Write a product description for [PRODUCT NAME].
Target customer: [DEMOGRAPHIC]
Key benefits: [3 BENEFITS]
Price point: [PRICE]
Tone: persuasive but honest
Include: a headline, 3 bullet points, and a closing CTA
Avoid: superlatives, empty claims, anything I cannot prove
```

**Why it works**: This prompt forces specificity. The constraint against superlatives produces copy that sounds credible instead of promotional. Businesses using this prompt report 15-25% higher conversion rates compared to generic descriptions.

### 2. Email Sequence That Converts
```
Write a 5-email nurture sequence for [BUSINESS TYPE].
Goal: convert a free trial user to paid subscriber.
Email 1: Welcome and quick win (day 0)
Email 2: Feature spotlight with use case (day 2)
Email 3: Social proof - customer success story (day 5)
Email 4: Overcome objection: [BIGGEST OBJECTION] (day 8)
Email 5: Urgency + offer (day 12)
Each email: under 200 words, one CTA, conversational tone.
```

**Why it works**: The five-email structure maps to proven SaaS conversion psychology. Specifying word count and single CTA prevents the bloated emails that kill engagement.

### 3. Social Media Content Calendar
```
Create a 7-day social media content plan for [BUSINESS].
Platforms: [LIST PLATFORMS]
Goals: [AWARENESS / ENGAGEMENT / TRAFFIC / SALES]
Content mix: 40% educational, 30% engagement, 20% promotional, 10% behind-the-scenes
For each post: platform, format, copy, hashtags, best time to post
```

**Why it works**: The content mix ratio prevents the common mistake of over-promoting. Educational content builds trust, engagement content grows reach, and promotional content converts -- but only if the ratio is right.

## Time-Saving Prompts

### 4. Meeting Summary to Action Items
```
Summarize the following meeting notes into:
1. Key decisions made (bullet points)
2. Action items with owners and deadlines
3. Open questions that need follow-up
4. Next meeting agenda items

Meeting notes:
[PASTE NOTES]
```

**Why it works**: This prompt eliminates the post-meeting scramble. Structured output means decisions do not get lost and action items have clear ownership.

### 5. Customer Support Response
```
Draft a support response for the following customer issue:
Issue: [DESCRIBE THE ISSUE]
Customer tone: [FRUSTRATED / CONFUSED / NEUTRAL]
Our policy: [RELEVANT POLICY]
Tone: empathetic, solution-focused, professional
Include: acknowledgment of the issue, clear next steps, timeline if applicable
Do not: blame the customer, use corporate jargon, make promises we cannot keep
```

**Why it works**: The "Do not" constraints are as important as the "Include" requirements. This prompt consistently produces responses that de-escalate frustrated customers.

### 6. Competitor Analysis Framework
```
Analyze [COMPETITOR NAME] based on publicly available information.
Structure:
- Product overview (what they sell, who they serve)
- Pricing model and tiers
- Strengths (what they do well)
- Weaknesses (where they fall short)
- Recent changes (last 90 days)
- Opportunity for us: where we can differentiate

Sources: website, social media, review sites, press releases
Format: markdown table where possible
```

**Why it works**: Structured competitive intelligence that would take an analyst 4 hours to compile. The "Opportunity for us" section makes it immediately actionable.

## Business Operations Prompts

### 7. Financial Summary
```
Based on the following data, create a financial summary:
[PASTE DATA OR DESCRIBE METRICS]

Include:
- Revenue breakdown by category
- Top 3 expenses
- Month-over-month trend (up/down/flat)
- Key ratio: revenue per customer
- One-line recommendation

Format: concise bullet points, no paragraphs
```

### 8. Blog Post Outline
```
Create a detailed outline for a blog post:
Topic: [TOPIC]
Target keyword: [SEO KEYWORD]
Audience: [WHO IS READING THIS]
Goal: [WHAT SHOULD THE READER DO AFTER READING]
Length: 1500 words

Structure:
- Title (include keyword, under 60 characters)
- Meta description (under 155 characters)
- H2 headings (5-7 sections)
- Key points per section (3-4 bullets each)
- CTA at the end
```

### 9. Hiring Job Description
```
Write a job description for [ROLE] at [COMPANY TYPE].
Include:
- Role summary (2-3 sentences, what the person will actually do)
- 5 core responsibilities (specific, measurable)
- Required qualifications (only what is truly required)
- Nice-to-have qualifications (be honest about what is optional)
- Compensation range: [RANGE]
- Why someone should want this job (honest, not generic)

Avoid: buzzwords, "rockstar/ninja," unrealistic requirements
```

### 10. Quarterly Business Review
```
Generate a quarterly business review template for [BUSINESS TYPE].
Cover:
- Revenue performance vs. target
- Customer acquisition and churn
- Top 3 wins this quarter
- Top 3 challenges
- Key metrics with quarter-over-quarter comparison
- Strategic priorities for next quarter
- Resource needs

Format: executive summary (5 bullet points) followed by detailed sections
```

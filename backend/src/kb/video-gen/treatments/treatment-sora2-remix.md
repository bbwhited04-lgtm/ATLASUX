## Key Takeaways
- Sora 2 supports video-to-video via **Remix** (being deprecated) and the newer **Edits** endpoint
- Remix transforms previously generated Sora videos using new text prompts while preserving motion/structure
- You can only remix videos that were generated by Sora (not arbitrary uploaded videos)
- Strength controls range from "subtle" to "mild" to "strong" to "custom" for how aggressively the remix modifies the source
- The Videos API exposes creation, extension, editing, and management of videos programmatically
- Sora 2 Pro provides cinematic-quality output with realistic physics and synchronized audio

## Which Models Support This
| Model | Capability | Status |
|-------|-----------|--------|
| Sora 2 | Remix (video-to-video) | Deprecated, use Edits |
| Sora 2 | Edits endpoint | Active, recommended |
| Sora 2 Pro | Enhanced quality + remix | Active |
| Sora 2 (via fal.ai) | Third-party remix API | Active |

## How It Works

### Remix Flow (Legacy)
1. Generate a base video via text-to-video or image-to-video
2. Use the `remix` endpoint with the `video_id` from step 1
3. Provide a new prompt describing the desired modifications
4. Set remix strength (subtle/mild/strong/custom)
5. The system preserves original motion and structure while applying style/content changes

### Edits Flow (Current)
1. Generate or reference an existing Sora-generated video
2. Use the `edits` endpoint for targeted modifications
3. Single, well-defined changes produce the best results
4. Smaller, focused edits preserve more original fidelity

### Additional Editing Tools (UI)
- **Recut**: Trim, extend, or regenerate specific video portions
- **Blend**: Create smooth transitions between two video clips
- **Loop**: Generate seamless loops from existing footage
- **Storyboard**: Combine multiple prompts/images into sequential scenes

## Code/API Examples

### Create a base video
```python
import openai

client = openai.OpenAI()

# Step 1: Generate base video
video = client.videos.create(
    model="sora-2",
    prompt="A golden retriever running through autumn leaves in a park",
    resolution="1280x720",
    duration=5
)

# Step 2: Poll for completion
status = client.videos.retrieve(video.id)
# ... poll until status is "completed"

# Step 3: Download the video
video_file = client.videos.download(video.id)
```

### Remix an existing video (Legacy endpoint)
```python
# Remix with style transfer
remixed = client.videos.remix(
    video_id=video.id,
    prompt="Same scene but in a cyberpunk neon-lit city at night",
    strength="strong"
)
```

### Edit a video (Current endpoint)
```python
# Targeted edit preserving original motion
edited = client.videos.edits.create(
    video_id=video.id,
    prompt="Change the dog's fur color to white",
)
```

### Via fal.ai (third-party)
```python
import fal_client

result = fal_client.submit("fal-ai/sora-2/video-to-video/remix", {
    "video_url": "https://...",
    "prompt": "Transform into anime style",
    "strength": 0.7
})
```

## Quality Considerations
- **Best for**: Style transfer, scene re-imagination, iterative creative refinement
- **Limitation**: Can only remix Sora-generated videos, not arbitrary uploads
- **Fidelity trade-off**: Higher remix strength = more creative freedom but less original preservation
- **Single edits work best**: Compound changes increase artifact risk
- **Resolution**: Supports 720x1280 and 1280x720 (must match source)
- **Cost**: ~$0.08-$0.20 per video depending on provider and quality tier
- **Audio**: Sora 2 generates synchronized audio natively

## Integration Notes for Atlas UX
- **Venny/Victor pipeline**: Use Sora 2 Edits endpoint for post-generation refinements (color correction, style shifts, scene modifications) after initial text-to-video generation
- **Iterative workflow**: Generate base video -> review -> apply targeted edits -> review -> finalize
- **Brand consistency**: Use the "subtle" strength setting when applying brand-specific color palettes to preserve original composition
- **Storyboard integration**: Leverage the Storyboard tool to chain multiple scenes with consistent style, then use Remix/Edits for per-scene refinement
- **API routing**: Route through OpenAI API directly for best quality; use fal.ai as fallback for cost-sensitive batch processing
- **Limitation awareness**: Since only Sora-generated videos can be remixed, the pipeline must generate the base video through Sora first -- cannot take user-uploaded footage and remix it

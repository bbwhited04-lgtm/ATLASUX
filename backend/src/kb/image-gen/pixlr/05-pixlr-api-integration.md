# Pixlr API and Integration Guide

## API Overview

Pixlr provides a Developer API and an official SDK for integrating its editing capabilities into third-party applications. The API is primarily designed for embedding the Pixlr editor into external platforms, not for programmatic AI image generation. This is an important distinction — the API gives your users access to the Pixlr editing interface within your app, but it does not expose direct endpoints for text-to-image generation or AI background removal as standalone API calls.

**Developer portal:** https://pixlr.com/developer/
**NPM package:** `@pixlrlte/pixlr-sdk`
**GitHub:** https://github.com/Pixlrcom/sdk
**API example project:** https://github.com/Pixlrcom/api-example

## Authentication

API access requires an API key and secret, generated from the Pixlr Developer section after creating a free account. Key points:

- Register at pixlr.com and navigate to the Developer section
- Generate your API Access Key and API Secret
- Always generate authentication tokens server-side — never expose the API secret to the client
- Free tier allows up to 1,000 API requests per month at no cost

## SDK Integration

The official `@pixlrlte/pixlr-sdk` npm package provides iframe-based embedding of the Pixlr editor into web applications.

### Installation

```bash
npm install @pixlrlte/pixlr-sdk
```

### Basic Usage

```typescript
import { PixlrSDK } from '@pixlrlte/pixlr-sdk';

// Initialize the SDK
const pixlr = new PixlrSDK({
  apiKey: 'YOUR_API_KEY',
  mode: 'embedded' // 'embedded' or 'http'
});

// Open an image in the embedded editor
pixlr.open({
  image: imageFileOrUrl,
  target: document.getElementById('pixlr-iframe'),
  onSave: (editedImage) => {
    // Handle the edited image
    console.log('Image saved:', editedImage);
  }
});
```

### SDK Modes

The SDK operates in two modes:

1. **Embedded mode** — Opens the Pixlr editor inside an iframe on your page. Communication between your app and the editor happens via `postMessage`. This is the recommended mode for web applications.

2. **HTTP mode** — Uses a form-based server-side API. The user is redirected to Pixlr's domain for editing, and the edited image is returned to a callback URL on your server.

### Communication Protocol

The embedded editor communicates with the host page through the browser's `postMessage` API. While it is technically possible to communicate directly with the Pixlr editor through internal protocols, the official SDK is strongly recommended for stability and forward compatibility.

## API Capabilities

### What the API Can Do

- **Embed the Pixlr editor** into your web application (Express or Editor)
- **Open images for editing** from URLs or file uploads
- **Receive edited images** via callback when the user saves
- **Customize the editor UI** (limited theming options)
- **Track editing sessions** for user activity logging

### What the API Cannot Do

- **No programmatic AI image generation** — You cannot send a text prompt via API and receive a generated image back. AI generation is only available through the interactive UI.
- **No headless background removal** — The AI background remover is not exposed as a standalone API endpoint. You cannot send an image and receive a background-removed version programmatically.
- **No batch processing API** — Batch operations are only available through the interactive editor UI.
- **No AI tool automation** — Generative fill, object removal, face swap, and other AI features are not accessible via API.

## Atlas UX Integration Assessment

### Viability for Atlas UX

**Rating: Limited / Not Recommended for core automation**

Pixlr's API is designed for embedding an interactive editor, not for automated image processing pipelines. For Atlas UX's use cases — where Lucy or the AI agents need to programmatically generate, edit, or process images without human interaction — Pixlr's API does not provide the necessary headless/programmatic capabilities.

### What Could Work

1. **Embedded editor for tenant users:** If Atlas UX wanted to offer tenants an in-app image editor for manually creating social media graphics or editing photos, the Pixlr SDK could embed the editor into the Atlas UX frontend. Tenants would get a familiar editing interface without leaving the platform.

2. **Image editing handoff:** The engine could prepare images (generated by another AI provider) and open them in an embedded Pixlr editor for the tenant to make final adjustments before publishing.

### What Would Not Work

1. **Automated image generation pipeline:** Lucy cannot use Pixlr to generate social media images, blog headers, or marketing materials programmatically. There is no text-to-image API endpoint.

2. **Automated background removal:** The engine cannot send product photos through Pixlr for automatic background removal as part of an e-commerce workflow.

3. **Batch processing automation:** Cannot programmatically batch-process tenant images through Pixlr's AI tools.

### Recommended Alternatives for Programmatic Use

For Atlas UX's automated image generation and editing needs, these platforms offer proper headless APIs:

| Need | Recommended Platform | API Available |
|------|---------------------|---------------|
| Text-to-image generation | OpenAI DALL-E 3, Stability AI, Replicate | Yes (REST API) |
| Background removal | remove.bg, PhotoRoom API, Clipdrop | Yes (REST API) |
| Image editing/compositing | Cloudinary, imgix | Yes (REST API) |
| Batch image processing | Cloudinary, Sharp (local) | Yes |
| All-in-one AI image pipeline | Replicate, Stability AI Platform | Yes (REST API) |

### Hybrid Approach

The most practical integration for Atlas UX would be a hybrid model:

- Use **programmatic APIs** (DALL-E, Stability AI, Cloudinary) for automated image generation and processing in the engine and agent workflows
- Offer **Pixlr's embedded editor** as an optional manual editing tool within the Atlas UX frontend for tenants who want to make fine-tuned adjustments to generated images

This gives tenants the best of both worlds: automated AI image generation for speed and scale, plus a professional browser-based editor for manual refinement when needed.

## Rate Limits and Quotas

- **Free API tier:** 1,000 requests per month
- **Paid API tiers:** Contact Pixlr for enterprise/B2B pricing (https://pixlr.com/lp/apib2b/)
- Rate limit details beyond the free tier are not publicly documented; enterprise agreements are negotiated directly

## Security Considerations

- API secrets must be stored server-side only (never in client-side code or environment variables exposed to the browser)
- The embedded editor runs in an iframe, providing same-origin policy isolation
- Images are transmitted to Pixlr's servers for AI processing — consider data privacy implications for tenant content
- No SOC 2 or HIPAA compliance documentation is publicly available from Pixlr

# Adobe Firefly API Integration

## API Overview

Adobe offers programmatic access to Firefly through **Firefly Services**, a suite of APIs available via the Adobe Developer Console. The API provides text-to-image generation, generative fill, generative expand, and other Firefly capabilities through REST endpoints. This is a production-grade API designed for enterprise integrations, not a lightweight developer sandbox.

## Getting Started

### Authentication Setup

1. Go to the [Adobe Developer Console](https://developer.adobe.com/console)
2. Create a new project
3. Select "Add API" and choose "Firefly - Firefly Services"
4. Select "OAuth Server-to-Server" as the authentication type (default)
5. Adobe generates a **Client ID** (API key) and **Client Secret**

### Token Management

Every API request requires an encrypted access token obtained from Adobe's Identity Management System (IMS):

```
POST https://ims-na1.adobelogin.com/ims/token/v3
Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials
&client_id=YOUR_CLIENT_ID
&client_secret=YOUR_CLIENT_SECRET
&scope=openid,AdobeID,firefly_api
```

**Critical details:**
- Access tokens expire after **24 hours**
- You must implement token refresh logic to maintain uninterrupted access
- Tokens should be cached and reused until near expiration
- All API calls include the token in the Authorization header: `Bearer <access_token>`

### Basic API Call

```
POST https://firefly-api.adobe.io/v3/images/generate
Authorization: Bearer <access_token>
x-api-key: <client_id>
Content-Type: application/json

{
  "prompt": "A professional headshot of a business person in a modern office, soft studio lighting",
  "n": 4,
  "size": {
    "width": 2048,
    "height": 2048
  },
  "contentClass": "photo",
  "styles": {
    "presets": ["photography"]
  }
}
```

The API returns URLs to the generated images (hosted temporarily on Adobe's CDN) along with Content Credentials metadata.

## Available API Endpoints

| Endpoint | Capability |
|----------|-----------|
| `/v3/images/generate` | Text-to-image generation |
| `/v3/images/fill` | Generative fill (inpainting with mask) |
| `/v3/images/expand` | Generative expand (outpainting) |
| `/v3/images/edit` | Image editing operations |
| `/v3/images/style-ref` | Apply style reference to generation |
| `/v3/images/structure-ref` | Apply structure/composition reference |

## SDK Support

Adobe provides an official SDK for Firefly Services:

```bash
npm install @adobe/firefly-services-sdk
```

The SDK handles token management, request formatting, and response parsing. It wraps the REST API with typed interfaces and automatic token refresh.

```typescript
import { FireflyClient } from '@adobe/firefly-services-sdk';

const client = new FireflyClient({
  clientId: process.env.ADOBE_CLIENT_ID,
  clientSecret: process.env.ADOBE_CLIENT_SECRET,
});

const result = await client.generateImages({
  prompt: "Modern minimalist logo for a plumbing company, clean vector style",
  n: 4,
  size: { width: 1024, height: 1024 },
  contentClass: "art",
});

// result.images[] contains URLs and metadata
```

## Content Credentials

Every image generated through the Firefly API includes **Content Credentials** (C2PA standard) embedded in the file metadata. This records:

- That the image was generated by AI
- Which Firefly model was used
- Timestamp of generation
- The generating application/API

Content Credentials cannot be disabled on Firefly-generated content. They persist through file format conversions and can be verified at [contentcredentials.org](https://contentcredentials.org). This is increasingly relevant for compliance, as regulations in the EU and proposed US legislation may require disclosure of AI-generated media.

For Atlas UX, Content Credentials provide a built-in audit trail for any AI-generated images used in marketing materials or client deliverables.

## Rate Limits and Quotas

- API access is metered through the same generative credit system as the web interface
- Enterprise API plans include dedicated credit allocations
- Rate limits are per-project, not per-token
- Concurrent request limits apply (typically 5-10 concurrent requests per project)
- 429 (Too Many Requests) responses should trigger exponential backoff

## Enterprise API Access

Enterprise customers get additional API capabilities:

- **Custom model training:** Train Firefly on brand-specific assets for on-brand generation
- **Batch processing:** Submit large batches of generation requests
- **Higher rate limits:** Increased concurrency and throughput
- **SLA guarantees:** Uptime and response time commitments
- **Dedicated support:** Technical account manager for integration

## Pricing for API Usage

API usage consumes generative credits from the associated Adobe account. There is no separate API pricing -- it draws from the same credit pool as web and application usage. For high-volume API integrations, enterprise pricing with custom credit allocations is recommended.

## Atlas UX Integration Considerations

### Feasibility

Firefly API integration into Atlas UX is technically feasible but introduces complexity:

1. **Authentication overhead:** OAuth Server-to-Server with 24-hour token rotation is more complex than a simple API key. The `credentialResolver.ts` pattern would need to store both Client ID and Client Secret per tenant, and a token management layer would need to handle refresh logic.

2. **Credit management:** Each tenant would need their own Adobe account with sufficient credits. Unlike per-API-call pricing (OpenAI, Stability), Firefly requires subscription-level planning.

3. **Commercial safety value:** For trade business marketing materials (the Atlas UX use case), Firefly's commercial safety guarantee is a genuine differentiator. A plumber's AI-generated ad created with Firefly carries zero copyright risk.

### Recommended Approach

If integrating Firefly into Atlas UX:

- Store Adobe Client ID and Client Secret in `tenant_credentials` (AES-256-GCM encrypted)
- Implement a token cache with automatic refresh in a dedicated `services/adobeFirefly.ts` service
- Use the SDK rather than raw REST calls for maintainability
- Position Firefly as the "safe for commercial use" option alongside faster/cheaper alternatives
- Route marketing material generation (ads, social posts, product photos) through Firefly
- Route creative exploration and concept work through Midjourney or DALL-E

### Alternative: Adobe Express Integration

For simpler integration, consider the Adobe Express Embed SDK, which provides a drop-in UI component with Firefly capabilities. This avoids API complexity but requires users to interact with Adobe's UI rather than a fully custom interface.

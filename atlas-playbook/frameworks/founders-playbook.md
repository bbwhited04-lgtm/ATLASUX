# Founder's Playbook — Atlas UX Edition

## Purpose
Week-by-week guide for Billy as solo founder. What to do, what to spend,
what milestones to hit, what decisions to make — from NOW through Month 12.

---

## STAGE 1: PRE-REVENUE (Now — You Are Here)

```
CURRENT STATE:
- Product: Built (Lucy, Mercer, SlackWorker functional)
- Landing page: atlasux.cloud (71 visits, 0 conversions)
- Revenue: $0
- Customers: 0
- Marketing: Multi-platform streaming
- Spend: ~$135/mo infrastructure + $520 recent one-time costs

PRIORITIES (in order):
1. Fix landing page to convert
2. Get Lucy demo phone number live
3. Start Mercer calling + short-form video
4. Get first 10 paying customers
5. Everything else waits
```

### Weeks 1-4: First 10 Customers
See `launch-engine-30day.md` for the detailed plan.

```
KEY ACTIONS:
□ Landing page conversion fix (demo, pricing, trust signals)
□ Mercer calling 50 trade businesses/day
□ 1-2 short-form videos/day
□ Personal onboarding for every trial user
□ Billy closing every deal personally

SPEND: $0 additional (content + outreach)
MILESTONE: 10 paying customers, ~$1,200 MRR
```

### Weeks 5-8: Validate & Optimize
```
KEY ACTIONS:
□ Collect testimonials from first 10 customers
□ Write first case study with real numbers
□ Optimize Mercer's script based on 4 weeks of data
□ Identify: which vertical converts best? Double down.
□ Start FAQ/help docs for common setup issues
□ Begin SEO: 2 blog posts/week targeting "AI receptionist for [trade]"

SPEND: ~$50/mo for tools (email marketing, analytics)
MILESTONE: 25 customers, ~$3,000 MRR, < 10% monthly churn
DECISION: Is the $39 Limited tier getting any traction? If not, kill it.
```

## STAGE 2: EARLY TRACTION (Months 3-6)

### Months 3-4: Get to 50 Customers
```
KEY ACTIONS:
□ Referral program live: "Give $25, get $25"
□ Mercer calling 100/day (expand list to new metro areas)
□ Content: weekly case study videos featuring real customers
□ Onboarding optimization: reduce friction, increase smart defaults
□ Weekly check-in with every customer (personal text from Billy)
□ Track unit economics religiously — are you making money per customer?

SPEND: ~$200/mo (tools + infra scaling)
MILESTONE: 50 customers, ~$6,000 MRR
DECISION: Ready for annual pricing option?
DECISION: Is Mercer's compliance airtight? (TCPA, DNC lists)
```

### Months 5-6: Prove Unit Economics
```
KEY ACTIONS:
□ Analyze: cost per customer by tier and call volume
□ Identify money-losing customers (high call volume on cheap tier)
□ Consider: usage caps, tier adjustments, or upsell paths
□ Build self-serve support (help docs, video guides)
□ Explore integration partnerships (Square, Jobber, ServiceTitan)
□ Start building email nurture sequence for trials

SPEND: ~$300/mo
MILESTONE: 100 customers, ~$12,000 MRR, > 50% gross margin
DECISION: Hire first person? (Support/sales — not engineering)
DECISION: Raise a pre-seed round?
```

## STAGE 3: GROWTH (Months 7-12)

### Months 7-9: Scale Acquisition
```
KEY ACTIONS:
□ Hire first person (customer support + sales assist)
□ Expand to 2nd and 3rd trade verticals
□ Mercer self-serve: let customers set up their own outbound campaigns
□ Partnership outreach: trade associations, vendor integrations
□ Consider: small paid ad budget ($500-1K/mo) on Facebook/Google

SPEND: ~$3,000-4,000/mo (first hire + marketing budget)
MILESTONE: 200 customers, ~$24,000 MRR
REVENUE COVERS: All costs + Billy's salary + first hire
```

### Months 10-12: Solidify & Plan
```
KEY ACTIONS:
□ Evaluate fundraising (if unit economics justify it)
□ Product: Zoom integration (if demand validated)
□ Product: Calendar integrations with trade-specific tools
□ Enterprise tier: pursue multi-location trade businesses
□ Build: customer dashboard v2 with better analytics
□ Culture: document processes before hiring person #2-3

MILESTONE: 300-500 customers, $36K-$60K MRR ($430K-$720K ARR)
DECISION: Fundraise or stay bootstrapped?
DECISION: Hire engineer? Or keep using AI-assisted development?
```

## LEGAL CHECKLIST (Do These NOW, Not Later)

```
□ Business entity: LLC filed in your state
□ Terms of Service on atlasux.cloud (before first customer)
□ Privacy Policy on atlasux.cloud (CCPA at minimum)
□ Call recording disclosure (see Agent 09 for two-party consent states)
□ TCPA compliance for Mercer (DNC list checking, time restrictions)
□ Stripe Terms of Service accepted
□ Supabase DPA (Data Processing Agreement) signed
□ Telephony provider terms reviewed
□ Trademark: "Atlas UX" — do a basic trademark search, file intent-to-use if clear
```

## FINANCIAL HEALTH CHECKS (Weekly)

```
□ Cash balance
□ MRR
□ Customer count (new, churned, net)
□ ARPU
□ Gross margin %
□ Biggest expense this week
□ Runway (months of cash at current burn)
```

## MENTAL HEALTH CHECKS (Real Talk)

```
□ Solo founder burnout is real — take at least 1 day off/week
□ 0 → 10 customers is the hardest stretch. It gets easier.
□ Every "no" from Mercer is data, not rejection
□ Compare yourself to yesterday, not to companies with 50 employees
□ Celebrate: first trial, first paid customer, first testimonial, first referral
□ If you're working 80 hours/week: you're not being productive, you're being anxious
```

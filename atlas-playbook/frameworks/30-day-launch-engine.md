# 30-Day Launch Engine

## Purpose
You have a product (or MVP). You need users. You have little or no budget.
This framework gets you from "product exists" to "people are paying for it"
in 30 days by forcing three decisions and executing relentlessly on them.

**This is not a marketing strategy. This is a launch engine.**
Strategy comes later, after you have 100 paying users and know what actually works.

---

## THE THREE DECISIONS (make these before Day 1)

### Decision 1: Positioning — Who is this for and why should they care?

```
POSITIONING STATEMENT (fill in every blank):

FOR _________________________ (specific person, not "everyone")
WHO _________________________ (specific situation they're in right now)
THIS IS _____________________ (category they already understand)
THAT ________________________ (one thing you do differently)
UNLIKE ______________________ (what they're currently using instead)
WE __________________________ (proof or mechanism that makes it credible)

EXAMPLES:
BAD:  "An AI-powered productivity platform for modern teams"
      (Who? What does it do? Why should I care?)

GOOD: "For freelance designers who waste 5+ hours/week on invoicing,
       this is an invoicing tool that auto-generates invoices from your
       Figma project names — unlike FreshBooks, it knows your projects
       because it connects directly to your design tools."

RULES:
□ If you can't fill in every blank, you don't know your customer well enough.
  Go back to Agent 02 (Discovery) and talk to 10 more people.
□ "Everyone" is not a customer. "Freelance designers in India billing
  international clients" is a customer.
□ The category must be one they already search for. "AI-powered productivity
  platform" is not a category anyone Googles. "Invoicing tool for freelancers" is.
□ The differentiator must be something they can VERIFY in 30 seconds.
  Not "better UX" (subjective). Not "AI-powered" (meaningless).
  YES: "generates invoices from your Figma projects automatically."
```

### Decision 2: Channel — Where do these people already gather?

```
PICK ONE. MAXIMUM TWO.

Your first 100 users will NOT come from "SEO" or "brand awareness" or
"content marketing." Those take 6-12 months. Your first 100 users come
from going WHERE THEY ALREADY ARE and saying something relevant.

CHANNEL SELECTION MATRIX:
Ask these questions. Score 1-5 for each channel. Pick the highest scorer.

| Question | Channel A | Channel B | Channel C |
|----------|-----------|-----------|-----------|
| Are my target users actively HERE right now? | | | |
| Can I participate WITHOUT paying? (or for <₹5K) | | | |
| Can I reach them DIRECTLY (not through an algorithm)? | | | |
| Can I get feedback (replies, DMs, comments)? | | | |
| Can I start TODAY (no approval, no setup time)? | | | |
| TOTAL | | | |

CHANNEL OPTIONS BY USER TYPE:

B2C (consumers):
□ Reddit: Find 3-5 subreddits where your users discuss the problem.
  NOT to spam — to genuinely help, then mention your solution when relevant.
□ WhatsApp/Telegram groups: Industry/interest groups. Ask to share, don't spam.
□ Instagram/TikTok: If your product is visual. Create 1 piece of content/day showing the problem→solution.
□ YouTube: "How to [solve problem]" videos. Product is the answer, not the pitch.
□ Local communities: Meetups, college groups, apartment WhatsApp groups (for hyperlocal).
□ Product Hunt: ONE launch. Plan it well. Good for tech-savvy early adopters.

B2B (businesses):
□ LinkedIn: Direct outreach to 10-20 decision-makers per day. Personal message, not InMail spam.
□ Cold email: 50 targeted emails/day. Personalized first line. Clear value prop. One ask.
□ Twitter/X: Engage with people who tweet about the problem. Build relationships, then pitch.
□ Industry Slack/Discord: Join communities where your buyers hang out. Help first, sell later.
□ Partnerships: Find someone who already has your audience. Offer to co-create something.
□ Events/conferences: Attend as a participant. Talk to people. Demo on your laptop.

MARKETPLACE / PLATFORM:
□ Supply side first: Personally onboard your first 50-100 suppliers/sellers.
□ Demand aggregation: Facebook groups, WhatsApp groups, Reddit threads where buyers congregate.
□ Strategic partnerships: Partner with 1-3 complementary businesses who serve the same audience.

YOUR ANTI-PATTERN DETECTOR:
⛔ "We'll do SEO and content marketing" → Takes 6+ months. Not a 30-day channel.
⛔ "We'll run Facebook/Google ads" → You don't know your CAC yet. You'll burn money learning.
⛔ "We'll go viral on social media" → Not a strategy. It's a wish.
⛔ "We'll get press coverage" → Press doesn't convert to users. It converts to vanity metrics.
⛔ "We'll partner with influencers" → Maybe later. First prove the product works with organic users.
✅ "I will personally reach 10 potential users per day in [specific channel]" → THIS is a 30-day strategy.
```

### Decision 3: Offer — What's the simplest thing I can ask them to do?

```
THE OFFER LADDER (start at the bottom, move up):

LEVEL 1 — FREE TRIAL / FREEMIUM (lowest friction):
"Try it free for 14 days. No credit card required."
Use when: Product value is obvious once they use it. Self-serve onboarding works.
Risk: Lots of signups, few conversions. Need strong activation flow.

LEVEL 2 — FREE + PAID UPGRADE (most common for SaaS):
"Free forever for basic use. ₹499/month for [specific premium feature]."
Use when: Free version delivers real value. Paid version is clearly worth more.
Risk: Users stay free forever. Need clear "aha moment" that makes them want more.

LEVEL 3 — PAID WITH MONEY-BACK GUARANTEE (good for trust):
"₹999/month. Try for 30 days. If it doesn't [specific outcome], full refund."
Use when: Product delivers measurable results. You're confident in the value.
Risk: Refund rate. But honest refund policy builds trust.

LEVEL 4 — PAID PILOT / DONE-FOR-YOU (B2B):
"We'll set it up for your team, run it for 2 weeks, show you the results. ₹25,000."
Use when: B2B, complex product, buyer needs hand-holding. Premium positioning.
Risk: Doesn't scale. But that's fine — you're learning, not scaling.

LEVEL 5 — WAITLIST / APPLICATION (scarcity):
"Apply to join our beta. We're accepting 50 companies this month."
Use when: You want high-quality users who are invested. Works for premium products.
Risk: Fewer signups. But higher quality.

YOUR OFFER FORMULA:
[Action] + [Timeframe] + [Outcome] + [Risk reversal]
Example: "Sign up today, and within 10 minutes you'll have your first
auto-generated invoice from Figma. Free for 14 days, cancel anytime."

RULES:
□ ONE offer. Not three pricing tiers. Not "contact us." ONE clear thing to do.
□ The offer must be completable in ONE session (don't make them "schedule a demo" unless B2B).
□ Include a deadline or scarcity element: "First 50 users get lifetime pricing."
□ Remove every possible friction point: No credit card for free trials. Social login. Pre-fill forms.
```

## THE 30-DAY PLAN

### Pre-Launch (Day -3 to Day -1): Set Up

```
DAY -3:
□ Finalize positioning statement (Decision 1)
□ Choose your channel (Decision 2)
□ Define your offer (Decision 3)
□ Write your one-liner: "[Product] helps [who] [do what] by [how]."

DAY -2:
□ Set up landing page with: Headline (one-liner), 3 bullet points (benefits, not features),
  screenshot/demo video, ONE call-to-action button (your offer), social proof if any
□ Tools: Carrd.co ($19/yr), Typedream (free), or your own domain
□ Set up analytics: PostHog or Mixpanel free tier. Track: Visit → Signup → Activation.
□ Set up email: Resend or Mailgun free tier. Welcome email auto-sends on signup.

DAY -1:
□ Test the entire flow yourself: Landing page → Signup → First value moment
□ Time it. If first value moment takes >5 minutes, simplify.
□ Prepare 5-10 personal outreach messages tailored to your channel.
□ Tell 10 friends/colleagues about launch (not to use it — to spread the word).
```

### Week 1 (Day 1-7): Manual Hustle — Get First 20 Users

```
DAILY ROUTINE (2-3 hours/day):
□ Spend 1 hour on outreach in your chosen channel (10+ people per day)
□ Spend 30 min responding to every reply, question, or signup
□ Spend 30 min observing: Who's signing up? Who's bouncing? What are they saying?

DAY 1-2: WARM NETWORK
□ Personal message to 50 people who might be your target user or know someone who is.
  Not "check out my app" — "I built something that solves [problem]. Would you try it
  and give me honest feedback? Takes 3 minutes."
□ Post in 2-3 relevant communities (Reddit, Slack, Discord, WhatsApp groups).
  NOT "I built this." YES: "I was frustrated by [problem], so I built [solution].
  Here's what I learned. Would love feedback."

DAY 3-5: COLD OUTREACH
□ 10-20 cold outreach messages per day in your primary channel.
□ Template (personalize the first line EVERY time):
  "Hey [Name], I saw you [specific thing they did/said about the problem].
  I built [product] that [specific solution to their specific problem].
  Would you try it? Free for [timeframe]. [Link]"
□ Track: Messages sent / Replies / Signups / Activations

DAY 6-7: FIRST FEEDBACK LOOP
□ By now you have 10-20 signups. Talk to EVERY ONE of them.
□ Call or DM: "Thanks for trying [product]. What almost stopped you from signing up?
  What's the one thing that would make you tell a friend about this?"
□ Fix the #1 thing that came up. Ship the fix same day if possible.

WEEK 1 TARGETS:
□ 50+ outreach messages sent
□ 15-20 signups
□ 5+ conversations with real users
□ 1 product fix based on feedback
```

### Week 2 (Day 8-14): Find What Works — Double Down

```
DAY 8-9: ANALYZE
□ Which messages got the most replies? (That's your messaging.)
□ Which channel drove the most signups? (That's your channel.)
□ What did users say in feedback? (That's your positioning refinement.)
□ How many users completed the first value action? (That's your activation rate.)

DAY 10-14: DOUBLE DOWN
□ Double your daily outreach in the channel that's working.
□ Refine your message to the version that got the best response rate.
□ Add social proof to landing page: "[X] people signed up this week" or
  a real quote from a real user's feedback.
□ Ask your happiest users: "Would you tell one person about this?"
  If they say yes: "Could you share this link with them? [tracking link]"
□ Post a "Week 1 learnings" in your community channel. Authenticity builds trust.

WEEK 2 TARGETS:
□ 100+ total outreach messages sent
□ 40-50 total signups
□ Activation rate measured (target: 30%+)
□ 2-3 users who would recommend to a friend
□ Positioning refined based on real user language
```

### Week 3 (Day 15-21): Activate and Convert — Get First Revenue

```
DAY 15-17: ACTIVATION FOCUS
□ Map the exact path from signup to first value moment.
□ Where do users drop off? Fix that ONE step.
□ Add an email sequence (3 emails over 5 days) nudging inactive signups:
  Email 1 (Day 0): "Welcome! Here's how to get started in 2 minutes."
  Email 2 (Day 2): "Did you try [key feature]? Here's what [user] got from it."
  Email 3 (Day 5): "Need help? Reply to this email — I'll personally walk you through it."

DAY 18-21: CONVERSION
□ If freemium: Personally message your most active free users.
  "You've used [product] X times this week. Would the paid version with
  [specific feature] be worth ₹499/month for you? I'll give you 50% off as an early user."
□ If paid trial: Day 10 of trial, message: "Your trial ends in 4 days.
  Anything I can help with? As a founding user, I'll lock in ₹X/month for life."
□ If B2B: Schedule a 15-min call with every trial user. Show value. Ask for the sale.

WEEK 3 TARGETS:
□ 60-80 total signups
□ Activation rate >30%
□ First 3-10 paying users
□ At least ₹5K-50K in revenue (proving willingness to pay)
□ One testimonial or case study from a real user
```

### Week 4 (Day 22-30): Systematize — Build Repeatability

```
DAY 22-25: SYSTEMATIZE WHAT WORKS
□ Your outreach message that works best → Save it as a template.
□ Your channel that converts best → Increase volume 50%.
□ Your onboarding emails → Automate them (Resend, Mailchimp, Loops).
□ Your activation fix → Monitor the metric daily.
□ Write down your FULL acquisition process: Where → Message → Landing → Signup → Activate → Convert.
  This is your Growth Playbook v0.1.

DAY 26-28: REFERRAL
□ Ask every paying user: "Know anyone else who has this problem?"
□ Build a simple referral mechanic: "Invite a friend → both get 1 month free."
□ Create a shareable asset: A template, checklist, or tool related to your product's value.
  Example: Invoice tool → "Free freelancer invoice template (made with [Product])."

DAY 29-30: RETROSPECTIVE AND PLAN
□ What worked? What didn't? What surprised you?
□ Key metrics: Total signups, activation %, paying users, revenue, CAC (time × your hourly value)
□ Decision: Do I have something people will pay for? (Yes → Scale. No → Pivot.)
□ Write your 60-day plan based on what you learned.

WEEK 4 TARGETS:
□ 100+ total signups
□ 10-30 paying users
□ Repeatable acquisition process documented
□ At least 1 referral-driven signup
□ Revenue: ₹10K-1L+ (or equivalent signal of willingness to pay)
```

```
Track daily. Review weekly. Decide monthly.

| Metric | Week 1 | Week 2 | Week 3 | Week 4 |
|--------|--------|--------|--------|--------|
| Outreach messages sent | | | | |
| Reply / response rate | | | | |
| Landing page visits | | | | |
| Signups | | | | |

| Active users (used product 2+ times) | | | | |
| Paying users | | | | |
| Revenue (₹) | | | | |
| User conversations had | | | | |
| Referrals received | | | | |
| NPS / "Would you recommend?" | | | | |

HEALTHY BENCHMARKS (Day 30):
□ Outreach → Reply rate: >10% (if <5%, your message is wrong)
□ Visit → Signup rate: >20% (if <10%, your landing page is wrong)
□ Signup → Activation rate: >30% (if <20%, your onboarding is wrong)
□ Activation → Paid rate: >5% (if <2%, your value or pricing is wrong)
□ NPS >30: People like it. Scale it.
□ NPS 0-30: Usable but not loved. Iterate before scaling.
□ NPS <0: Something is fundamentally wrong. Pivot.
```

# Universal Checklists — Atlas UX

## Purpose
Quick-reference checklists for recurring decisions and processes.
Don't reinvent the wheel every time.

---

## PRE-LAUNCH CHECKLIST

```
PRODUCT:
□ Lucy answers calls reliably (test 20+ calls across scenarios)
□ Appointment booking works end-to-end
□ Dashboard shows call logs and notifications
□ Signup → Lucy live in < 15 minutes
□ Billing works (trial start, trial end, payment, cancellation)
□ Phone forwarding setup documented and tested

LANDING PAGE:
□ Hero message clear in 5 seconds
□ Demo phone number live and working
□ Pricing visible and simple
□ Trust signals present (your photo, story)
□ CTA clear and above the fold
□ Mobile-responsive
□ Page loads in < 3 seconds
□ Analytics tracking (who visits, from where)

LEGAL:
□ Terms of Service published
□ Privacy Policy published
□ Call recording consent in Lucy's greeting
□ TCPA compliance for Mercer (DNC list, time restrictions)
□ Stripe terms accepted
□ Business entity (LLC) filed

OPERATIONS:
□ Support email set up
□ Monitoring/alerting active (Lucy uptime)
□ Backup plan if Lucy goes down (calls forward to Billy)
□ Daily tracking sheet ready
```

## NEW CUSTOMER ONBOARDING CHECKLIST

```
□ Welcome text sent (personal, from Billy)
□ Business setup complete (name, type, hours)
□ Lucy greeting customized
□ Calendar connected
□ Phone forwarding active
□ Test call completed successfully
□ Owner knows how to access dashboard
□ Owner knows how to reach support
□ Day 3 check-in scheduled
□ Day 7 call stats sent
□ Day 14 trial conversion follow-up scheduled
```

## CONTENT CREATION CHECKLIST (Per Video)

```
□ Hook in first 3 seconds (question, stat, or scenario)
□ Problem clearly stated (missed call = lost money)
□ Solution shown (Lucy answers)
□ Specific to a trade vertical (salon, plumber, etc.)
□ CTA at the end (link in bio, try Lucy free)
□ Under 60 seconds for short-form
□ Captions/subtitles added (many watch on mute)
□ Posted on: TikTok, IG Reels, YouTube Shorts, Facebook
□ Engaged with first 10 comments within 1 hour
```

## WEEKLY BUSINESS REVIEW CHECKLIST

```
METRICS:
□ MRR (up or down from last week?)
□ New customers this week
□ Churned customers this week
□ Trial-to-paid conversion rate
□ Mercer: calls made, demos booked, trials started
□ Content: videos posted, total views, link clicks
□ Landing page: visits, signups, conversion rate
□ Lucy: total calls handled, appointments booked, issues flagged
□ Support: tickets received, response time, satisfaction

ACTIONS:
□ Follow up with all trial users
□ Check in with any customer who had an issue
□ Review Lucy call transcripts for quality (sample 5-10)
□ Review Mercer outcomes — adjust script if needed
□ Plan next week's content
□ Update tracking sheet
```

## MERCER CAMPAIGN LAUNCH CHECKLIST

```
□ Target list built (business name, phone, vertical, location)
□ DNC list cross-referenced — all flagged numbers removed
□ Script written and tested (10 test calls)
□ Calling hours set (8am-9pm local time only)
□ Caller ID shows real, callable Atlas UX number
□ AI disclosure included if required by state
□ Opt-out mechanism working ("don't call me again" → instant DNC)
□ Tracking active (calls, interested, demos, trials)
□ Max 3 attempts per business with 48-hour spacing
□ Billy available for warm transfers during campaign hours
```

## PRICING CHANGE CHECKLIST

```
□ Financial model updated with new pricing
□ Unit economics still positive? (Gross margin > 50%)
□ Existing customers: grandfathered or migrated? (Grandfather first 50)
□ Landing page updated
□ Stripe products/prices updated
□ Mercer's script updated
□ All marketing materials updated
□ Email sent to existing customers (if affected)
□ FAQ updated
□ Tested: can a new user sign up at new price?
```

## INCIDENT RESPONSE CHECKLIST

```
□ Acknowledge: "We know about the issue, working on it"
□ Diagnose: What's broken? Since when? How many affected?
□ Mitigate: Immediate fix or workaround (forward calls to owner's cell)
□ Fix: Root cause resolution
□ Verify: Confirm fix works with test calls
□ Communicate: Update affected customers
□ Document: What happened, why, what we changed
□ Prevent: What stops this from happening again?
```

## MONTHLY FINANCIAL REVIEW CHECKLIST

```
□ Total revenue this month
□ Total costs this month (infrastructure, telephony, AI, tools)
□ Gross margin %
□ Cash balance
□ Runway (months at current burn)
□ Cost per customer (total costs ÷ customers)
□ Cost per call (total variable costs ÷ total calls)
□ Any customers costing more than they pay? (Flag for upsell)
□ Any unexpected charges? (API overages, etc.)
□ Update revenue projection spreadsheet
```

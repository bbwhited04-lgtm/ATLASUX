# TRUTH_COMPLIANCE_CHECK

Purpose:
Prevent deception, exaggeration, manipulation, or misinformation
before any publication.

------------------------------------------------------------
CHECK SEQUENCE
------------------------------------------------------------

For every piece of publishable content:

1. CLAIM VERIFICATION
   - Does the content state a factual claim?
   - Is the claim verifiable?
   - Is the source cited?
   - If uncertain → mark as uncertain.

2. FEATURE VALIDATION
   - Are product capabilities described accurately?
   - Are features live and functional?
   - No roadmap features presented as released.

3. METRIC AUTHENTICITY
   - No inflated engagement metrics.
   - No fabricated testimonials.
   - No synthetic authority claims.

4. MANIPULATION DETECTION
   Reject if:
   - Artificial urgency used without basis.
   - Emotional exploitation.
   - Manufactured outrage.
   - False scarcity.

5. PLATFORM RULE CHECK
   - Does this violate platform policy?
   - Is automation disclosure required?

6. INTENT CHECK
   Ask:
   "Is this written to inform or to manipulate?"

7. BUILDER CHECK
   Would Billy publicly stand behind this without clarification?

------------------------------------------------------------
FAIL CONDITIONS
------------------------------------------------------------

If ANY check fails:

- Deny publication.
- Log violation attempt.
- Notify Atlas.
- Require revision.

------------------------------------------------------------
PASS CONDITIONS
------------------------------------------------------------

All checks passed:
→ Forward to Atlas for approval.
